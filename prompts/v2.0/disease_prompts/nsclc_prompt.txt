### 2. disease_stage
**OPTIONAL** - Can be null, but try hard to infer before accepting null.

**Valid values (in order of preference):**
- `"Early-stage resectable"` - Stage I-IIIA resectable, surgical candidates
- `"Early-stage unresectable"` - Stage III unresectable/locally advanced (concurrent chemoRT setting)
- `"Early-stage"` - Fallback when early-stage but resectability unclear
- `"Metastatic"` - Stage IV or metastatic disease

**Preference hierarchy for early-stage:**
1. If resectable is mentioned or patient is surgical candidate → `"Early-stage resectable"`
2. If unresectable/locally advanced/concurrent chemoRT → `"Early-stage unresectable"`
3. If neither specified but clearly early-stage → `"Early-stage"` (fallback)

**When to infer:**
- Trial context: FLAURA → Metastatic, CheckMate-816 → Early-stage resectable, PACIFIC → Early-stage unresectable, ADAURA → Early-stage resectable
- Treatment line: 1L/2L+ → Metastatic; Adjuvant/Neoadjuvant → Early-stage resectable; Consolidation → Early-stage unresectable
- Drug indication: Most targeted therapies imply metastatic unless adjuvant context

**When null is appropriate:**
- General biomarker testing questions without disease context
- MOA questions about a drug without indication specified
- Questions spanning multiple stages

**Triggers:**
| Trigger keywords | → disease_stage |
|------------------|-----------------|
| "mNSCLC", "metastatic", "stage IV", "advanced NSCLC" | → `"Metastatic"` |

### 29. answer_length_pattern
Pattern of answer option lengths. **DEFAULT: "Uniform"**

**PURPOSE**: This field is for **question quality auditing**. We want to identify questions where answer length could be a test-taking cue.

**CRITICAL: Most questions are "Uniform".** Use "Variable" ONLY when answer options are fundamentally different TYPES.

**Valid values:**
- `"Uniform"` - **DEFAULT (90%+ of questions).** All options are conceptually similar in type and scope. Minor length differences due to drug names, combo regimens, or acronym expansion do NOT make it Variable.
- `"Variable"` - **RARE.** Only when options are fundamentally different types — typically correlates with "Heterogeneous" distractors.
- `"Correct longest"` - Correct answer is dramatically longer than all distractors (flaw indicator)
- `"Correct shortest"` - Correct answer is dramatically shorter than all distractors

**Key insight**: Variable length pattern is almost always tied to Heterogeneous distractors.

### 30. distractor_homogeneity
Whether answer options are structurally similar (same TYPE) or fundamentally different.

**PURPOSE**: This field is for **question quality auditing**. Heterogeneous distractors can indicate poor question design.

**Valid values:**
- `"Homogeneous"` - **DEFAULT for well-written questions.** All options are the **same TYPE** — all drug names, all regimens, all clinical phrases about similar concepts. If distractors are similar enough that they don't raise question quality concerns, use Homogeneous.
- `"Heterogeneous"` - Options are **fundamentally different TYPES** that mix categories. Examples: drug names mixed with dosing instructions, treatment options mixed with monitoring recommendations.

**Decision rule**: Ask "Are these all the same kind of answer?" If yes → Homogeneous. If mixing different categories → Heterogeneous.

### Item Writing Flaw Fields (31-36)
These 6 fields are **boolean** (true/false). Set to `true` if the flaw is present, `false` if not.

### 31. flaw_absolute_terms
`true` if answer options contain testwise cues like "always", "never", "all", "none" that make them less likely to be correct.

### 32. flaw_grammatical_cue
`true` if stem grammar reveals the answer (e.g., "an" before vowel-starting answer, singular/plural mismatch).

### 33. flaw_implausible_distractor
`true` if any distractor is obviously wrong and would not be chosen by a competent oncologist.

### 34. flaw_clang_association
`true` if the correct answer shares unusual words or phrases with the stem that could guide to the answer without knowledge.

### 35. flaw_convergence_vulnerability
`true` if multiple options share elements and the correct answer contains all shared elements (test-taking strategy vulnerability).

### 36. flaw_double_negative
`true` if there's a negative stem combined with negative answer option creating confusion.

## Important Guidelines

1. **disease_type is almost always null** - Only tag when "Squamous" or "Leptomeningeal" is explicitly stated
2. **Molecular alterations go in biomarker** - EGFR, ALK, ROS1, KRAS, PD-L1, etc. are tagged as biomarker, NOT disease_type
3. **Infer biomarkers from drug/trial context** - osimertinib → EGFR mutation, alectinib → ALK rearrangement
4. **EGFR exon 20 is distinct** - Different from classical EGFR, different treatments (amivantamab, not osimertinib)
5. **Stage III considerations** - Unresectable stage III gets concurrent chemoRT + consolidation, not metastatic treatment
6. **Consider the answer** - Molecular subtype may be revealed in the correct answer
7. **Tag content from the CORRECT answer only**: Only tag treatments, biomarkers, and other entities that appear in the CORRECT answer, NOT the distractors (incorrect options). The distractors are wrong/not applicable by definition.
   - **Standard questions ("Which of the following...")**: Tag ONLY entities from the correct answer
   - **Negative lead-ins ("All EXCEPT", "Which is NOT")**: Tag the INCORRECT options instead, as they represent the true/factual statements
   - **Compound answers ("A and B", "All of the above")**: Tag all components that make up the correct answer
8. **Fundable drugs only** - Don't tag chemotherapy or radiation
9. **Infer from context** - Trials, drugs, and clinical context can imply stage, biomarker, and treatment line
