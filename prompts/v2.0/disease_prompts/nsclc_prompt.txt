## NSCLC-Specific Topic Guidance

### Topic Disambiguation

**Treatment indication vs Treatment selection:**
- "Which patients are candidates for osimertinib?" → **Treatment indication** (eligibility/criteria)
- "Which TKI is preferred for this patient?" → **Treatment selection** (choosing among options)

**Safety profile vs AE management vs Prophylaxis:**
- "What are common toxicities of amivantamab?" → **Safety profile** (awareness, knowing what to expect)
- "Patient develops Grade 3 rash on amivantamab, what should you do?" → **AE management** (active management of occurring AE)
- "What prophylactic measures should be taken before amivantamab infusion?" → **Safety profile** (prevention/monitoring is part of safety awareness)

**Clinical efficacy vs Study design:**
- "What was the PFS in FLAURA2?" → **Clinical efficacy** (trial results)

## NSCLC-Specific Rules

### disease_type (Critical Difference from Other Diseases)

**NSCLC uses `null` for most cases.** Molecular alterations go in `biomarker`, NOT `disease_type`.

| disease_type Value | When to Use |
|-------------------|-------------|
| `null` | **DEFAULT** - Use for most NSCLC questions |
| `"Squamous"` | When squamous histology is stated or clearly implied (see rules below) |

**⚠️ CRITICAL: Squamous Histology Must ALWAYS Be Tagged**
If the question stem, activity name, or clinical context mentions "squamous", "squamous cell", "squamous NSCLC", or "squamous cell carcinoma of the lung", you **MUST** tag `disease_type: "Squamous"`. This is NOT optional.

**Why squamous matters for NSCLC:**
- Squamous histology changes treatment selection (e.g., bevacizumab contraindicated, pemetrexed less effective)
- Different 1L chemo-IO regimens: KEYNOTE-407 (squamous) vs KEYNOTE-189 (non-squamous)
- Some targeted therapies have histology-specific data
- Squamous patients are less likely to harbor EGFR/ALK/ROS1 alterations

**Tag squamous in these scenarios:**
- "metastatic squamous NSCLC" → disease_type: `"Squamous"`
- "squamous cell carcinoma of the lung" → disease_type: `"Squamous"`
- KEYNOTE-407 context (squamous-specific trial) → disease_type: `"Squamous"`
- Carboplatin-paclitaxel-pembrolizumab (squamous regimen) → disease_type: `"Squamous"` if stated

**Leptomeningeal disease is NOT a disease_type.** Tag it as `metastatic_site_1: "Leptomeningeal disease"` instead. Leptomeningeal disease describes a site of metastatic spread, not a molecular or histologic subtype.

**Why this matters:**
- EGFR-mutated NSCLC → disease_type: `null`, biomarker_1: `"EGFR mutation"`
- ALK-positive NSCLC → disease_type: `null`, biomarker_1: `"ALK"`
- KRAS G12C NSCLC → disease_type: `null`, biomarker_1: `"KRAS G12C"`

This differs from breast cancer where HER2+ IS the disease_type.

### treatment_line

**⚠️ ABSOLUTE PROHIBITION: "Newly diagnosed" is NEVER a valid treatment_line for NSCLC.**
NSCLC is a solid tumor — use line-of-therapy (1L, 2L+) for metastatic, and setting (Neoadjuvant, Adjuvant, Perioperative) for early-stage. "Newly diagnosed" is only used for hematologic malignancies. If the patient is newly diagnosed with metastatic NSCLC, the treatment_line is `"1L"`.

**Early-Stage (Resectable):**
- `"Neoadjuvant"` - Systemic therapy given BEFORE surgery to shrink the tumor (e.g., CheckMate-816: nivolumab + chemo → surgery)
- `"Adjuvant"` - Systemic therapy given AFTER surgery to reduce recurrence (e.g., ADAURA: osimertinib after resection; IMpower010: atezolizumab after resection)
- `"Perioperative"` - Systemic therapy given BOTH before AND after surgery as a planned sequence (e.g., KEYNOTE-671: pembrolizumab + chemo → surgery → pembrolizumab; CheckMate-77T: nivolumab + chemo → surgery → nivolumab)

**How to distinguish:**
- If the trial/treatment involves therapy ONLY before surgery → `"Neoadjuvant"`
- If the trial/treatment involves therapy ONLY after surgery → `"Adjuvant"`
- If the trial/treatment involves therapy before AND after surgery as a single protocol → `"Perioperative"`
- If the question discusses adjuvant therapy in the context of a perioperative trial, still tag `"Perioperative"` (the trial design defines the setting)

**Unresectable Stage III:**
- `"Consolidation"` - After concurrent CRT (e.g., PACIFIC: durvalumab after CRT; LAURA: osimertinib after CRT for EGFR+)

**Metastatic:**
- `"1L"` - First-line metastatic (newly diagnosed metastatic = 1L)
- `"2L+"` - Second-line or later (post-progression on prior therapy)
- `"Maintenance"` - Continuation of one agent after completing induction

### Biomarker Tagging Rules

**Actionable Genomic Alterations (AGAs):**
Tag the specific molecular alteration in `biomarker_1`:
- `"EGFR mutation"` - Classic sensitizing mutations (exon 19 del, L858R)
- `"EGFR exon 20 insertion"` - **Distinct from classic EGFR** (different treatment pathway - see below)
- `"ALK"` (ALK rearrangement/fusion)
- `"ROS1"` (ROS1 rearrangement)
- `"KRAS G12C"` (specific mutation)
- `"BRAF V600E"` (specific mutation)
- `"MET exon 14 skipping"` - **Distinct from c-MET overexpression** (see below)
- `"c-MET overexpression"` - **Distinct from MET exon 14 skipping** (see below)
- `"RET"` (RET fusion)
- `"NTRK"` (NTRK fusion)
- `"HER2 mutation"` (HER2 mutation in NSCLC, NOT amplification - see below)

**CRITICAL DISTINCTION: EGFR exon 20 insertion vs Classic EGFR Mutations**
These are SEPARATE biomarker values with completely different treatment pathways:
- Classic EGFR (exon 19 del, L858R) → biomarker_1: `"EGFR mutation"` → osimertinib, amivantamab + lazertinib
- EGFR exon 20 insertion → biomarker_1: `"EGFR exon 20 insertion"` → amivantamab (+/- chemo), mobocertinib, sunvozertinib, BDTX-1535
- Do NOT lump them together. If question says "EGFR exon 20 insertion", tag exactly that.

**CRITICAL DISTINCTION: MET exon 14 Skipping vs c-MET Overexpression**
These are SEPARATE biomarker values with completely different treatment contexts:
- `"MET exon 14 skipping"` = A driver mutation (AGA). Treated with MET TKIs (capmatinib, tepotinib). These are the PRIMARY treatment.
- `"c-MET overexpression"` = An expression biomarker, often a resistance mechanism or predictive biomarker for c-MET-targeting ADCs (telisotuzumab vedotin). Not a classic AGA.
- If question discusses MET as a resistance mechanism after EGFR TKI, consider `resistance_mechanism: "MET amplification"` and `biomarker_1: "c-MET overexpression"`.

**HER2 Mutation in NSCLC:**
HER2 mutations in NSCLC (distinct from HER2 amplification in breast cancer) are treated with:
- Trastuzumab deruxtecan (T-DXd) - ADC
- Zongertinib - HER2-selective TKI
- Sevabertinib - HER2-mutant targeted TKI
- For generalized "HER2-mutant NSCLC" questions, infer ALL approved/available HER2-targeting agents at STARTDATE.

**⚠️ CRITICAL: Biomarker Priority in Post-Progression Settings**
When a patient has a known driver mutation (e.g., EGFR mutation) BUT the question is about treatment directed at a DIFFERENT biomarker after progression, prioritize the **treatment-selection biomarker** (the one guiding the current treatment decision) in `biomarker_1`:

| Scenario | biomarker_1 (PRIMARY) | biomarker_2 (BACKGROUND) |
|----------|----------------------|-------------------------|
| EGFR-mutated patient, post-osimertinib, now c-MET overexpressing → telisotuzumab vedotin | `"c-MET overexpression"` | `"EGFR mutation"` |
| EGFR-mutated patient, post-osimertinib, now HER2 amplified → T-DXd | `"HER2 amplification"` | `"EGFR mutation"` |
| ALK+ patient, post-lorlatinib, MET amplification detected | `"MET amplification"` | `"ALK"` |

**Rule:** `biomarker_1` = the biomarker driving the CURRENT treatment decision. Background driver mutations go in `biomarker_2` (or `prior_therapy` / `resistance_mechanism` as appropriate). Do NOT let the original driver mutation crowd out the actionable biomarker for the question being asked.

**"No AGA" Convention:**
When a question discusses immunotherapy eligibility for patients WITHOUT actionable alterations:
- biomarker_1: `"No AGA"` (No Actionable Genomic Alteration)
- This indicates the patient is immunotherapy-eligible

**"Pending" Biomarker:**
When molecular testing results are described as pending or awaited:
- biomarker_1: `"Pending"` - indicates testing ordered but results not yet available
- Common in questions about initial treatment decisions while awaiting genomic results

**PD-L1 Expression:**
Tag PD-L1 status when relevant to treatment selection:
- `"PD-L1 >=50%"` (high expression, pembrolizumab monotherapy eligible)
- `"PD-L1 1-49%"` (intermediate expression)
- `"PD-L1 <1%"` or `"PD-L1 negative"` (low/negative)

**⚠️ CRITICAL: Drug Targets are NOT Biomarkers**
Do NOT confuse drug targets with biomarkers. They serve different purposes:
- **Biomarker** = A patient characteristic used for treatment SELECTION (tested in the patient)
- **Drug target** = The molecular entity a drug binds to (property of the drug)

| Scenario | biomarker_1 | drug_target_1 |
|----------|------------|---------------|
| EGFR-mutated patient on osimertinib | `"EGFR mutation"` | `"EGFR"` |
| Patient getting pembrolizumab for PD-L1 >=50% | `"PD-L1 >=50%"` | `"PD-1"` |
| Telisotuzumab vedotin for c-MET overexpressing tumor | `"c-MET overexpression"` | `"c-MET"` |
| T-DXd for HER2-mutant NSCLC | `"HER2 mutation"` | `"HER2"` |
| DLL3-targeting therapy in SCLC | Do NOT tag DLL3 as biomarker (unless testing question) | `"DLL3"` |

**Rule:** When a question discusses a targeted therapy, the patient's molecular status goes in `biomarker`, and the drug's molecular target goes in `drug_target`. These are often related but NOT interchangeable.

## NSCLC Edge Cases

### Edge Case 1: Squamous vs Non-Squamous
- Tag `disease_type: "Squamous"` ONLY when squamous histology affects treatment (e.g., bevacizumab contraindicated)
- For most questions, leave `disease_type: null` even if "squamous" is mentioned

### Edge Case 2: Resistance Mutations
- T790M after EGFR TKI → biomarker_1: `"T790M"`, prior_therapy_1: `"Prior EGFR TKI"`, resistance_mechanism: `"T790M mutation"`
- C797S after osimertinib → biomarker_1: `"C797S"`, prior_therapy_1: `"Prior osimertinib"`, resistance_mechanism: `"C797S mutation"`

### Edge Case 3: MET Amplification as Resistance
- MET amplification arising during EGFR TKI → resistance_mechanism: `"MET amplification"`, NOT biomarker_1

### Edge Case 4: Combination Immunotherapy
- Nivolumab + ipilimumab → treatment_1: `"nivolumab"`, treatment_2: `"ipilimumab"`, drug_class_1: `"Immune checkpoint inhibitor"`, drug_target_1: `"PD-1"`, drug_target_2: `"CTLA-4"`

### Edge Case 5: Leptomeningeal Disease
- Tag as metastatic_site_1: `"Leptomeningeal disease"` (NOT disease_type)
- Often involves intrathecal therapy or osimertinib (CNS-penetrant)
- May also tag special_population_1: `"CNS involvement"`

### Edge Case 6: ADCs After TKI Progression
When a question discusses treatment options after progression on targeted therapy (e.g., post-osimertinib):
- The ADC landscape is biomarker-directed:
  - TROP2-directed: datopotamab deruxtecan
  - HER3-directed: patritumab deruxtecan
  - c-MET-directed: telisotuzumab vedotin (requires c-MET overexpression)
  - HER2-directed: trastuzumab deruxtecan (requires HER2 mutation)
- Tag `prior_therapy_1: "Prior EGFR TKI"` or `"Prior osimertinib"` as context
- Tag `treatment_line: "2L+"` for post-TKI settings

### Edge Case 7: Generalized Biomarker-Driven Questions
When a question asks about treatment for a biomarker-defined population WITHOUT naming specific drugs (e.g., "What are the treatment options for HER2-mutant NSCLC?"):
- **Infer ALL approved/available drugs** for that biomarker at the STARTDATE
- Example: "HER2-mutant NSCLC treatment" → treatment_1: `"trastuzumab deruxtecan"`, treatment_2: `"zongertinib"`, treatment_3: `"sevabertinib"` (if available at STARTDATE)
- This is critical for CME funding attribution - the question is about the entire drug class
- Use the Fundable Treatments table above and your knowledge to determine which drugs were available

### Edge Case 8: Resistance Mechanisms
Common resistance patterns and how to tag them:
| After | Resistance | Tags |
|-------|-----------|------|
| 1st/2nd gen EGFR TKI | T790M | resistance_mechanism: `"T790M mutation"` |
| Osimertinib | C797S | resistance_mechanism: `"C797S mutation"` |
| Osimertinib | MET amplification | resistance_mechanism: `"MET amplification"` |
| Osimertinib | HER2 amplification | resistance_mechanism: `"HER2 amplification"` |
| Osimertinib | Histologic transformation (SCLC) | resistance_mechanism: `"Histologic transformation"` |

## CRITICAL: Tagging from Correct Answer Only

**Standard questions:** Tag treatments, biomarkers, toxicities, and other clinical entities ONLY from the CORRECT answer. Do NOT tag entities from incorrect options (distractors).

**Why this matters:** Distractors exist to test knowledge. Tagging a distractor drug would misrepresent the question's educational focus. A question where the correct answer is osimertinib is about osimertinib, not about the three wrong drugs listed alongside it.

**Example:**
Q: "Which targeted therapy is preferred for 1L EGFR-mutated metastatic NSCLC?"
A: Osimertinib (correct), B: Erlotinib, C: Gefitinib, D: Afatinib
- treatment_1: `"osimertinib"` ONLY
- Do NOT tag erlotinib, gefitinib, or afatinib

**Compound questions (EXCEPTION):** When the question format is "All of the following EXCEPT", "Which combination of A and B", or the answer involves multiple entities:
- Tag entities from ALL relevant answer options
- **IMPORTANT: "A + B" compound answer labels do NOT necessarily refer to the first two answer options.** The letters A and B in a compound answer are just labels. Read the actual content of each option to determine what is being combined. For example, if answer choices are: A) Osimertinib, B) Erlotinib, C) Pembrolizumab, D) A and C — then D means Osimertinib + Pembrolizumab, NOT A + B (Osimertinib + Erlotinib).

## Core Classification (4 fields)

### 1. topic
**REQUIRED** - Every question must have exactly one topic. Never null.

| **Safety profile** | Knowing WHAT AEs to expect or recognize — awareness, anticipation. "What are the common AEs?", "Which symptoms should you monitor?" |

| **Multidisciplinary care** | Team-based care, referrals, care coordination |

| **Radiation therapy management** | Radiation planning, dosing, sequencing with systemic therapy (SBRT, CRT, brain radiation) |
| **Surgical management** | Surgical approach, resectability, surgical decision-making, timing relative to systemic therapy |
| **Disparities in care** | Racial/ethnic disparities, inequities in access/outcomes |

## Multi-value Fields (15 fields)

### 5-9. treatment_1 through treatment_5
**OPTIONAL** - Only tag fundable drugs of pharmaceutical interest.

**CRITICAL RULES:**
- DO NOT tag: chemotherapy, radiation, surgery, hormonal backbone (fulvestrant, letrozole when combined with fundable drug)
- For combinations with one fundable drug, tag ONLY the fundable drug
- Put each drug in a separate field (treatment_1, treatment_2, etc.)
- Use `null` for unused slots

**Fundable drug categories:**
- Targeted therapies (TKIs, mAbs)
- Antibody-drug conjugates (ADCs)
- Immunotherapies (checkpoint inhibitors, CAR-T, bispecifics)
- Novel agents in clinical trials


### treatment_1-5: Critical Tagging Rules

**Rule 1: Subject vs Comparator**
Tag ONLY the drug that is the SUBJECT of the teaching point, not comparators.

EXAMPLE:
Q: "Which AE differentiates Drug A from Drug B and Drug C?"
A: "Specific toxicity"
✓ treatment_1: Drug A (subject being differentiated)
✗ DO NOT TAG: Drug B, Drug C (just comparators)

**Rule 2: Treatment as Clinical Context**
Tag treatments when they provide CONTEXT for interpreting a clinical finding.

**Rule 3: Drug Class → Specific Approved Drugs**
When the correct answer is a drug CLASS, you MUST tag all approved specific drugs.
Use the disease-specific drug table above to map class → individual drugs.

### 10-14. biomarker_1 through biomarker_5

**DO NOT put in biomarker:**
- **Driver mutations already in disease_type** → If EGFR mutation defines disease_type, don't also tag biomarker "EGFR" (redundant). Exception: questions about biomarker TESTING
- **Drug targets** → Use `drug_target` instead (e.g., PD-L1 as target for pembrolizumab)
- **Resistance mutations** (e.g., T790M, C797S) → Use `resistance_mechanism` instead

**OPTIONAL** - Tag when clinically relevant AND not redundant with disease_type.

**REDUNDANCY RULE:** If biomarker is already captured in disease_type, use null
- Example: If disease_type is "EGFR-mutated", don't also tag biomarker: "EGFR mutation"

**EXCEPTION:** If topic is "Biomarker testing", you MAY tag the biomarker being tested

**Common values (use these when applicable, but may use other values if confident):**
- Predictive: `"PD-L1"`, `"EGFR mutation"`, `"ALK"`, `"HER2"`, `"BRCA1/2"`, `"MSI-H"`, `"TMB"`, `"PIK3CA"`, `"ESR1"`
- Prognostic: `"Ki-67"`, `"Oncotype DX"`, `"MammaPrint"`
- Resistance: `"T790M"`, `"ESR1"`, `"C797S"`

### 15-19. trial_1 through trial_5
**CONDITIONAL** — Tag behavior depends on topic:

**Always tag** when a trial name is **explicitly mentioned** in the question stem or answer choices (any topic).

**Inference ALLOWED** only when topic is:
- "Clinical efficacy" — Infer from drug + indication + efficacy context
- "Study design" — Infer from drug + indication + study methodology context

**Inference NOT ALLOWED** for all other topics (Treatment selection, Treatment indication, AE management, Biomarker testing, Mechanism of action, Patient selection, Supportive care, Monitoring/follow-up). For these topics, use null unless the trial is explicitly named in the stem or answer choices.

**EXCEPTION — Landmark trial inference for highly specific clinical scenarios:**
When the clinical context maps to ONE specific landmark trial with no ambiguity, inference IS allowed regardless of topic. These scenarios are so defining that the trial is inseparable from the indication:
- Stage III unresectable EGFR+ NSCLC + post-CRT consolidation → trial: "LAURA", treatment: "Osimertinib"
- Adjuvant osimertinib after resected EGFR+ NSCLC → trial: "ADAURA", treatment: "Osimertinib"
- 1L osimertinib + chemo in EGFR+ advanced NSCLC → trial: "FLAURA2", treatment: "Osimertinib"
This exception applies ONLY when the scenario-to-trial mapping is unambiguous (1:1). If multiple trials could fit, do NOT infer.

## Group F: Question Format/Quality (13 fields)

This section evaluates both the educational level AND the structural quality of the question for internal QA.

### 54. cme_outcome_level
**OPTIONAL** - Moore's level of CME outcome.

**Valid values:**
- `"3 - Knowledge"` - Question tests recall/recognition of information
- `"4 - Competence"` - Question tests application of knowledge to a clinical scenario

**CRITICAL RULE:** If the clinical vignette is NOT REQUIRED to answer the question, it's **Level 3 (Knowledge)**, not Level 4.

**Test:** Could someone answer this question correctly WITHOUT reading the patient details?
- YES → `"3 - Knowledge"` (the vignette is just context/decoration)
- NO → `"4 - Competence"` (must apply knowledge to the specific patient)

**How to determine:**
- **Level 3 (Knowledge):** "What is the mechanism of action of X?", "Which biomarker predicts response to Y?"
- **Level 4 (Competence):** Patient vignette asking "What is the best treatment for this patient?", case-based decision making

### 55. data_response_type
**OPTIONAL** - Type of response the question elicits (content type).

**Valid values:**
- `"Numeric"` - Answer is a specific number (e.g., "What was the ORR?")
- `"Qualitative"` - Answer is descriptive (e.g., "Which best describes the mechanism?")
- `"Comparative"` - Answer compares options (e.g., "Which is preferred?")
- `"Boolean"` - Answer is Yes/No (e.g., "Is X approved for Y?")

### 56. stem_type
**OPTIONAL** - Format of the question stem.

**Valid values:**
- `"Clinical vignette"` - Patient case scenario (tests application)
- `"Direct question"` - Straightforward question without patient context (tests recall)
- `"Incomplete statement"` - "The mechanism of X is..." format

### 57. lead_in_type
**OPTIONAL** - Type of question lead-in.

**Valid values:**
- `"Standard"` - "Which of the following...", "What is the most..."
- `"Negative (EXCEPT/NOT)"` - "All of the following EXCEPT...", "Which is NOT..."
- `"Best answer"` - "What is the BEST...", "most appropriate", "most likely"
- `"True statement"` - "Which statement is TRUE...", "Which is correct..."

### 58. answer_format
**OPTIONAL** - Structure of answer options.

**Valid values:**
- `"Single best"` - Standard single correct answer format
- `"Compound (A+B)"` - "A and B", "Both A and C" as options
- `"All of above"` - "All of the above" is an answer option
- `"None of above"` - "None of the above" is an answer option
- `"True-False"` - True/False format

### 59. answer_length_pattern
**REQUIRED** - Every question must have an answer length pattern. Never null.

**Valid values:**
- `"Uniform"` - All answer options approximately same length
- `"Variable"` - Mix of short and long answers
- `"Correct longest"` - Correct answer is noticeably longest
- `"Correct shortest"` - Correct answer is noticeably shortest

**MANDATORY DEFAULT RULE:**
If `distractor_homogeneity` = "Homogeneous", then `answer_length_pattern` MUST be "Uniform". Period.
The ONLY exception is an EGREGIOUS length disparity (single word vs full sentence). If you have to think about it, it's "Uniform".

**Rule of thumb:** Homogeneous distractors = Uniform answer length. Always. Do not overthink this.

**When to override to "Variable":**
Override ONLY when there is an EXTREME disparity in answer length or detail level:
- One option is a single word ("Osimertinib") and another is a full clinical sentence with multiple concepts ("Continue carboplatin-pemetrexed with close monitoring for progression and repeat CT every 3 cycles")
- One option is 3-4 words and another is 15+ words with multiple distinct clinical actions

**When to keep "Uniform" despite minor length differences:**
- All options are treatment regimens with similar structure, even if one has slightly more detail (e.g., "Osimertinib + amivantamab + lazertinib" vs "Osimertinib")
- All options are clinical management choices (e.g., "Continue therapy", "Switch treatment", "Supportive care"), even if one is 2-3 words longer

**"Uniform"** = All options are CONCEPTUALLY similar in complexity:
- All single drug names: "Osimertinib" / "Erlotinib" / "Afatinib" / "Gefitinib" → **Uniform**
- All short phrases: "Continue therapy" / "Switch treatment" / "Supportive care" → **Uniform**
- All treatment regimens: "Pembrolizumab + carboplatin + pemetrexed" / "Nivolumab + ipilimumab + chemo" / "Atezolizumab + bevacizumab + chemo" → **Uniform**

**"Variable"** = Options differ in TYPE or COMPLEXITY with EGREGIOUS disparity:
- Mix of single words and detailed explanations: "Osimertinib" / "Continue current carboplatin-pemetrexed regimen and monitor for progression with serial imaging every 2 cycles" → **Variable**

**"Correct longest/shortest"** = The CORRECT answer is noticeably longer/shorter than ALL distractors

### 60. distractor_homogeneity
**REQUIRED** - Every question must have a distractor homogeneity rating. Never null.

**Valid values:**
- `"Homogeneous"` - All distractors from same category
- `"Heterogeneous"` - Mixed distractor types

**"Homogeneous"** = All distractors are the SAME TYPE:
- All drug names: "Osimertinib" / "Erlotinib" / "Afatinib" / "Gefitinib" → **Homogeneous**
- All treatment regimens/combinations: "Pembrolizumab + carboplatin + pemetrexed" / "Nivolumab + ipilimumab + chemo" / "Atezolizumab + bevacizumab + chemo" → **Homogeneous** (all are treatment options, even if regimen complexity varies)
- All clinical actions: "Continue therapy" / "Dose reduce" / "Discontinue" / "Switch to 2L" → **Homogeneous**
- All trial names: "FLAURA" / "MARIPOSA" / "ADAURA" / "CROWN" → **Homogeneous**
- All biomarkers: "EGFR" / "ALK" / "ROS1" / "KRAS G12C" → **Homogeneous**

**KEY RULE:** If ALL answer options are treatments (whether single drugs, regimens, or combinations), they are **Homogeneous** — even if one option has more drugs than another. The test is: are they all the same CATEGORY of answer (treatments)? If yes → Homogeneous.

**"Heterogeneous"** = Distractors are DIFFERENT TYPES:
- Treatment + Safety + Mechanism: "Osimertinib" / "Neutropenia occurs in 50%" / "Targets EGFR via covalent binding" → **Heterogeneous**
- Mixed categories: drug name vs. dosing schedule vs. monitoring plan
- Substantively different clinical assertions requiring independent evaluation: "This patient should receive chemoimmunotherapy" / "Molecular testing is not indicated" / "Surgery should be considered first" / "Best supportive care is appropriate" → **Heterogeneous**

**Decision rule:** If options are interchangeable in format (just swapping one entity for another), use "Homogeneous". If each option must be evaluated on its own clinical merits, use "Heterogeneous".

### Item Writing Flaws (61-66)

These 6 fields are **boolean** (true/false). Set to `true` if the flaw is present, `false` if not.

### 61. flaw_absolute_terms
**BOOLEAN** - Answer options contain absolute terms.

Set `true` if any answer option contains: "always", "never", "all", "none", "only", "must", "cannot"

**Why it matters:** Test-wise examinees know absolutes are usually wrong.

### 62. flaw_grammatical_cue
**BOOLEAN** - Stem grammar reveals the answer.

Set `true` if:
- Stem ends with "a" or "an" and only one option starts with correct vowel/consonant
- Stem uses singular but some options are plural (or vice versa)
- Subject-verb agreement issues that eliminate options

**Why it matters:** Grammar matching guides to correct answer without content knowledge.

### 63. flaw_implausible_distractor
**BOOLEAN** - Contains obviously wrong options.

Set `true` if any distractor is clearly wrong to anyone with basic clinical knowledge (e.g., "Homeopathy" as treatment option for cancer, a drug from completely wrong class).

**Why it matters:** Reduces effective choices and inflates guessing success.

### 64. flaw_clang_association
**BOOLEAN** - Answer shares unusual words/phrases with stem.

Set `true` if the correct answer contains a distinctive word or phrase that also appears in the stem (word repetition beyond common clinical terms).

**Why it matters:** Word repetition guides to correct answer without actual knowledge.

### 65. flaw_convergence_vulnerability
**BOOLEAN** - Correct answer combines elements from multiple options.

Set `true` if:
- Multiple options share common elements
- The correct answer has the most overlapping elements with other options
- Test-taker could reason to correct answer by finding the "most complete" option

**Why it matters:** Allows reasoning to answer without content knowledge.

### 66. flaw_double_negative
**BOOLEAN** - Negative stem combined with negative option.

Set `true` if:
- Stem contains "NOT", "EXCEPT", "LEAST likely", etc. AND
- One or more answer options contain negative phrasing

**Why it matters:** Increases cognitive load, tests reading comprehension rather than knowledge.

# Universal Tagging Rules

These rules apply to ALL diseases. Disease-specific rules in the disease module take precedence when there is a conflict.

---

## Core Principles

### 1. Use Null Appropriately
- Use `null` when a value cannot be determined with confidence
- Use `null` for unused multi-value slots (e.g., treatment_3 through treatment_5 when only 2 drugs)
- **Never guess** - if uncertain, use `null`

### 2. Consider the Correct Answer
- The correct answer often reveals context not in the question
- Subtype, treatment line, and stage may be implied by the answer
- Use answer content to inform your tagging

### 3. Topic Definitions (Apply Consistently)

| Topic | Key Trigger | Example |
|-------|-------------|---------|
| **Treatment selection** | Choosing WHICH drug/regimen (patient vignette + "which therapy") | "A 55-year-old woman... which regimen is most appropriate?" |
| **Treatment indication** | Whether patient MEETS CRITERIA — eligibility, candidacy | "Which patients are candidates for X?", "Is this patient eligible?" |
| **Clinical efficacy** | Trial results, response rates, survival | "What was the ORR in TRIAL-X?" |
| **Safety profile** | Knowing WHAT AEs to expect/recognize (awareness) | "What are the common AEs?", "Which symptoms should you monitor?" |
| **AE management** | What to DO about an AE already occurring | "Patient develops pneumonitis, what is the next step?" |
| **Biomarker testing** | Testing, companion diagnostics | "Which patients should be tested for X?" |
| **Mechanism of action** | How drug works | "What is the target of X?" |
| **Study design** | Enrollment, endpoints, trial structure | "Which patients were enrolled in TRIAL-X?" |

### 4. Tag from CORRECT answer only
Only tag treatments, biomarkers, toxicities, and other clinical entities from the CORRECT answer. Do NOT tag entities from incorrect options (distractors).
- **Standard questions ("Which of the following...")**: Tag ONLY entities from the correct answer
- **Compound questions ("Which combination...", "All of the following EXCEPT...")**: Tag entities from ALL relevant options
- **Example**: If Q asks "Which toxicity is most associated with X?" and A: CRS, B: ICANS, C: Neuropathy, D: Hepatotoxicity with correct answer A, ONLY tag CRS as toxicity, NOT the distractors.

---

## Treatment Tagging Rules

### Fundable Drugs Only
Tag ONLY drugs of pharmaceutical interest for CME funding:
- Targeted therapies (TKIs, mAbs)
- ADCs
- Immunotherapies (checkpoint inhibitors, CAR-T, bispecifics)
- Novel agents

### DO NOT Tag
- **Chemotherapy agents**: carboplatin, paclitaxel, docetaxel, capecitabine, doxorubicin, etc.
- **Hormonal backbone**: fulvestrant, letrozole, anastrozole, tamoxifen (when combined with fundable drug)
- **Radiation therapy**
- **Surgical procedures**
- **Supportive care**: G-CSF, antiemetics, steroids (unless explicitly the focus)

### Combination Therapy
- For combinations with ONE fundable drug: tag only the fundable drug
  - "ribociclib + letrozole" → treatment_1: "ribociclib" (letrozole not tagged)
  - "pembrolizumab + chemotherapy" → treatment_1: "pembrolizumab"
- For combinations with MULTIPLE fundable drugs: tag each in separate slots
  - "trastuzumab + pertuzumab" → treatment_1: "trastuzumab", treatment_2: "pertuzumab"

### Drug Class Expansion
When question mentions a drug class without specific agent:
- Expand to all approved drugs in that class (as of question STARTDATE)
- Put each drug in a separate treatment slot
- Example: "Checkpoint inhibitor" for NSCLC → treatment_1-5: pembrolizumab, nivolumab, atezolizumab, durvalumab, cemiplimab

**IMPORTANT: Drug class expansion applies to AE management and Safety profile topics too.**
When a question asks about managing an AE or safety profile of a drug CLASS (not a specific drug), infer and tag the specific approved drugs using disease context and STARTDATE:
- Example: "Patient on a checkpoint inhibitor develops immune-related pneumonitis" → Infer approved checkpoint inhibitors for NSCLC at STARTDATE
- Example: "EGFR TKI-associated skin rash management" → Infer approved EGFR TKIs (osimertinib, erlotinib, etc.)
- The causative drug class is clinically relevant — funders need to know WHICH specific drugs are associated with the AE

## Biomarker Redundancy Rules

### Core Rule
**DO NOT tag a biomarker if it is already captured in disease_type**

| If disease_type is... | Don't tag biomarker: |
|-----------------------|---------------------|
| `"EGFR-mutated"` | `"EGFR mutation"` |
| `"HER2+"` | `"HER2"` |
| `"ALK-positive"` | `"ALK"` |
| `"MSI-H/dMMR"` | `"MSI-H"` |
| `"BRCA-mutated"` | `"BRCA1/2"` |

### Exception: Biomarker Testing Topic
If topic is `"Biomarker testing"` and question is about testing for a specific biomarker, you MAY tag that biomarker even if it's in disease_type.

### Non-Redundant Biomarkers
Always OK to tag biomarkers that are ADDITIONAL to disease_type:
- PIK3CA (in HR+/HER2- breast cancer → tag as biomarker)
- ESR1 (in HR+/HER2- → tag as biomarker)
- PD-L1 (in any disease → tag as biomarker)
- TMB (in any disease → tag as biomarker)

## Trial Inference Rules

### When to Infer Trial Names
**ALLOWED** for topics: `"Clinical efficacy"`, `"Study design"`
- Use drug + indication + population to confidently identify the trial
- Must have high confidence - if uncertain, use `null`

**NOT ALLOWED** for all other topics: `"Treatment selection"`, `"Treatment indication"`, `"AE management"`, `"Biomarker testing"`, `"Mechanism of action"`, `"Patient selection"`, `"Supportive care"`, `"Monitoring/follow-up"`, `"Safety profile"`
- These ask "which drug to use" or about non-efficacy concepts — not "what did the trial show"
- Only tag trial if explicitly named in the question stem or answer choices

### Inference Requirements
All of these must be clear to infer a trial:
1. Specific drug(s)
2. Specific indication/disease state
3. Specific treatment line or setting
4. Clinical efficacy or study design context

**CRITICAL: If you infer a trial, you MUST also tag the treatment.**
It is WRONG to infer trial_1: "FLAURA2" but leave treatment_1 as null. If you know the trial, you know the drug. Tag both.

## Group A-F Field Guidelines

### Group A: Treatment Metadata
- **drug_class**: Use when discussing a class of drugs (e.g., "EGFR TKI")
- **drug_target**: Use when discussing molecular target
- **prior_therapy**: Only tag when explicitly mentioned as prior treatment
- **resistance_mechanism**: Only tag when explicitly discussed

### Group B: Clinical Context
- **metastatic_site**: Only tag sites explicitly mentioned
- **symptom**: Only tag symptoms mentioned in vignette
- **special_population**: Tag patient characteristics that affect treatment (elderly, organ dysfunction)
- **performance_status**: Only tag if explicitly stated (ECOG, KPS, fit/unfit)

### Group C: Safety/Toxicity
- **toxicity_type**: Tag specific toxicities discussed
- **toxicity_organ**: Tag organ system if toxicity focus
- **toxicity_grade**: Tag grade if mentioned (Grade 3, Grade >=3)

### Group D: Efficacy/Outcomes
- **efficacy_endpoint**: Tag ONLY for "Clinical efficacy" topic OR when endpoint explicitly named in stem
- **outcome_context**: Tag analysis type (primary endpoint, subgroup, interim)
- **clinical_benefit**: Tag nature of benefit (statistically significant, clinically meaningful)

### Group E: Evidence/Guidelines
- **guideline_source**: Tag ONLY when guideline body is explicitly named in stem/answers (NCCN, ASCO, ESMO)
- **evidence_type**: Tag study design (Phase 3 RCT, real-world evidence)
- **trial_phase**: Tag phase if mentioned

### Group F: Question Format
- **cme_outcome_level**:
  - Level 3 (Knowledge): Tests recall/recognition
  - Level 4 (Competence): Tests application to clinical scenario
- **data_response_type**:
  - Numeric: Answer is a number
  - Qualitative: Answer is descriptive
  - Comparative: Answer compares options
- **endpoint_type**: Only for questions about trial endpoints

## Special Situations

### AE as Secondary Condition
When question asks about managing an AE (e.g., MDS from therapy):
- Keep the PRIMARY disease being treated as the disease context
- Use topic = "AE management" or "Safety profile"
- DO NOT change disease_state to the AE/complication

### Pan-Tumor Questions
For questions not specific to one disease:
- disease_type: `null`
- disease_stage: `null` (unless clearly stated)
- Focus on the treatment/biomarker being discussed

### Unknown Trial
If a trial is mentioned but you don't recognize it:
- Still tag the trial name as stated
- Tag what you can infer from context
- Use `null` for fields you cannot determine
