## CRITICAL: Compound Answer Handling

When the correct answer references options by letter/number (e.g., "B or C", "A and C", "Options 1 and 3"):

1. **DO NOT assume option order matches the original question** - The options you receive may be in a different order than the actual test question.

2. **Deduce the correct answer from clinical reasoning** - Read the substance of each option and determine which ones make sense as the correct answer based on the clinical scenario.

3. **Tag based on the deduced correct content** - Once you identify what the compound answer actually refers to, tag those treatments/concepts.

## Group A: Core Fields (20 fields)

### 1. topic
**REQUIRED** - Every question must have exactly one topic. Never null.

| Topic | Use When... |
|-------|-------------|
| **Treatment selection** | Choosing WHICH drug/regimen to use for a patient (patient vignette + "which therapy") |
| **Treatment indication** | Whether a patient MEETS CRITERIA for a treatment - eligibility, candidacy, **approved indications** |
| **Clinical efficacy** | Response rates, survival outcomes, clinical trial results, "key findings" |
| **Safety profile** | Knowing WHAT adverse events to expect or recognize |
| **AE management** | What to DO about an AE already occurring - hold drug, dose-reduce, supportive care |
| **Prophylaxis** | Knowing HOW TO PREVENT toxicities BEFORE they occur - infection prevention, TLS prophylaxis |
| **Biomarker testing** | Genetic testing, molecular profiling, cytogenetics, flow cytometry |
| **Mechanism of action** | Drug targets, pathways, how a drug works |
| **Diagnosis** | Diagnostic workup, classification, staging workup |
| **Prognosis** | Risk stratification, prognostic scoring systems |
| **Study design** | Trial design, patient enrollment criteria, endpoints |
| **Multidisciplinary care** | Team-based care, supportive care coordination |
| **Shared decision-making** | Patient-provider communication, discussing treatment options |
| **Disparities in care** | Racial/ethnic disparities, access issues |
| **Barriers to care** | Obstacles to treatment (cost, access, adherence) |

## Group G: Question Format/Quality (15 fields)

### 56. cme_outcome_level
**Values:** `"3 - Knowledge"` or `"4 - Competence"`

Apply this test:
1. **Can someone answer correctly using ONLY memory/recall?**
   - YES → `"3 - Knowledge"` (definitions, mechanisms, facts, adverse events, trial results)
   - NO → Continue to step 2

2. **Does answering require applying knowledge to a specific patient scenario?**
   - YES → `"4 - Competence"` (patient vignette with decision-making, treatment selection for a specific case)
   - NO → `"3 - Knowledge"`

**Key insight:** A question with a patient vignette is NOT automatically Level 4. If the vignette just sets up a factual question (e.g., "What is the mechanism of this drug?"), it's still Level 3.

### 57. data_response_type
**Values:** "Numeric", "Qualitative", "Comparative", "Boolean"

### 58. stem_type
**Values:** "Clinical vignette", "Direct question", "Incomplete statement"

### 59. lead_in_type
**Values:** "Standard", "Negative (EXCEPT/NOT)", "Best answer"

### 60. answer_format
**Values:** "Single best", "Compound (A+B)", "All of the above", "None of the above"

### 61. answer_length_pattern
**Values:** "Uniform", "Variable", "Correct longest"
- Use "Correct longest" ONLY when the correct answer is SUBSTANTIALLY longer (>50% longer) than distractors

### 62. distractor_homogeneity
**Values:** "Homogeneous", "Heterogeneous"
- "Homogeneous" = All options are same category (all drugs, all doses, all symptoms)
- "Heterogeneous" = Options mix different categories

### 63-68. Flaw fields (all Boolean: true/false)
- `flaw_absolute_terms` - Contains "always", "never", "all", "none"
- `flaw_grammatical_cue` - Grammar reveals answer
- `flaw_implausible_distractor` - Obviously wrong option
- `flaw_clang_association` - Similar-sounding words as cue
- `flaw_convergence_vulnerability` - Common element across options
- `flaw_double_negative` - Confusing double negatives

### 69. answer_option_count
**Computed field** - Count of answer options (2-5)

### 70. correct_answer_position
**Computed field** - Position of correct answer (A, B, C, D, E)
